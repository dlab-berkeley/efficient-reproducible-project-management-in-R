Organizing files

  • Note that setwd() is not reproducible outside the authors local machine. Instead, learn to use here().

  • Do not save .RData when you quit R and dont load. Using Bash, add the following line in .bash_profile: alias R='R --no-save --no-restore-data' (More information on invoking R by W. N. Venables and D. M. Smith (R Core Team))

(Required) Project organization

Read Software Carpentrys article on project management in RStudio.

Tips for file organization

  • See Good Enough Practices in Scientific Computing by PLOS

  • You can create directories on your computer from R using dir.create function.

  1. Create a project directory
# Create a project directory 
# .. subdirectory 
# I assume that your current working directory is advanced-data-wrangling-in-R. Check this by running `getwd()`

dir.create("../project")
## Warning in dir.create("../project"): '../project' already exists
  1. Treat raw data as read only (raw data should be RAW!)
dir.create(here::here("project", "data"))
## Warning in dir.create(here::here("project", "data")): '/home/jae/efficient-
## reproducible-project-management-in-R/project/data' already exists
  1. Separate read-only data from processed data
dir.create(here::here("project", "processed_data"))
## Warning in dir.create(here::here("project", "processed_data")): '/home/jae/
## efficient-reproducible-project-management-in-R/project/processed_data' already
## exists
  1. Treat generated outputs as disposable
dir.create(here::here("project", "outputs"))
## Warning in dir.create(here::here("project", "outputs")): '/home/jae/efficient-
## reproducible-project-management-in-R/project/outputs' already exists
  1. Separate function definition and application
dir.create(here::here("project", "functions"))
## Warning in dir.create(here::here("project", "functions")): '/home/jae/efficient-
## reproducible-project-management-in-R/project/functions' already exists
