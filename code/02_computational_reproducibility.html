  • How it works

here() function shows whats the top-level project directory.

here::here()
## [1] "/home/jae/efficient-reproducible-project-management-in-R"
  • Build a path including subdirectories
here("project", "outputs")
## [1] "/home/jae/efficient-reproducible-project-management-in-R/project/outputs"

  • How here defines the top-level project directory. The following list came from the here package repository).

    • Note that we already defined an RStudio Project!

  • Is a file named .here present?

  • Is this an RStudio Project? Literally, can I find a file named something like foo.Rproj?

  • Is this an R package? Does it have a DESCRIPTION file?

  • Is this a remake project? Does it have a file named remake.yml?

  • Is this a projectile project? Does it have a file named .projectile?

  • Is this a checkout from a version control system? Does it have a directory named .git or .svn? Currently, only Git and Subversion are supported.

  • If theres no match then use set_here() to create an empty .here file.
